ELMA365 On-Premises / Подготовка инфраструктуры

Подготовка инфраструктуры

Инфраструктура — это необходимые для работы приложения ELMA365 компоненты. В зависимости от задач различаются подходы и требования к организации инфраструктуры ELMA365 On-Premises.

Рассмотрим примеры:

  • инфраструктуры для ознакомления;
  • инфраструктуры DEV/TEST-окружений;
  • инфраструктуры PROD-окружения;
  • отказоустойчивой инфраструктуры PROD-окружения.

Начало внимание

Вы можете выбрать один из вариантов организации инфраструктуры.

Конец внимание

Инфраструктура для ознакомления

Этот вариант организации инфраструктуры подходит для ознакомительных целей и не требует повышенной надёжности компонентов хранения данных.

  1. Окружение с доступом в интернет.

Характеристики:

  • один физический сервер или виртуальная машина;
  • установка с доступом в интернет;
  • встроенные компоненты хранения данных.

Подробнее о том, как развернуть этот вариант инфраструктуры, читайте в статье «Установка ELMA365 Standard».

  1. Окружение без доступа в интернет.

Характеристики:

  • один физический сервер или виртуальная машина для всех компонентов;
  • установка без доступа в интернет;
  • встроенное хранилище образов Kubernetes;
  • встроенные компоненты хранения данных.

Подробнее о том, как развернуть этот вариант инфраструктуры, читайте в статье «Офлайн-установка ELMA365 Standard».

Инфраструктура DEV/TEST-окружений

Этот вариант организации инфраструктуры подходит для окружений разработки (DEV) и тестирования (TEST) и не требует повышенной надёжности компонентов хранения данных.

  1. Окружение с доступом в интернет.

Характеристики:

  • один физический сервер или виртуальная машина для всех компонентов;
  • установка с доступом в интернет;
  • встроенные компоненты хранения данных.

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Standard, читайте в статье «Установка ELMA365 Standard».

О том, как развернуть этот вариант инфраструктуры для ELMA365 Enterprise, читайте в статьях:

  • «Kubernetes кластер»;
  • «Подготовка встроенных баз данных»;
  • «Установка ELMA365 Enterprise в Kubernetes».
  1. Окружение без доступа в интернет.

Характеристики:

  • один физический сервер или виртуальная машина для ELMA365 On-Premises и компонентов хранения данных;
  • один физический сервер или виртуальная машина для хранилища образов Kubernetes;
  • установка без доступа в интернет;
  • локальное хранилище образов Kubernetes;
  • встроенные компоненты хранения данных.

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Standard, читайте в статьях:

  • «Установка Harbor»;
  • «Загрузка образов ELMA365»;
  • «Установка ELMA365 Standard».

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Enterprise, читайте в статьях:

  • «Установка Harbor»;
  • «Загрузка образов ELMA365»;
  • «Загрузка образов Deckhouse»;
  • «Kubernetes кластер»;
  • «Подготовка встроенных баз данных»;
  • «Установка ELMA365 Enterprise в Kubernetes».

Инфраструктура PROD-окружения

Вариант организации инфраструктуры для окружений эксплуатации (PROD).

  1. Окружение с доступом в интернет.

Характеристики:

  • один физический сервер или виртуальная машина для ELMA365 On-Premises, компонентов проксирования и хранения данных;
  • три физических сервера или виртуальных машины для внешних баз данных;
  • установка с доступом в интернет;
  • встроенные компоненты хранения данных RabbitMQ, Redis;
  • внешние компоненты хранения данных MongoDB, PostgreSQL, S3 MinIO;
  • встроенный прокси-сервер HAProxy для проксирования входящих запросов к внешнему S3-хранилищу.

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Standard, читайте в статьях:

  • «MongoDB»;
  • «PostgreSQL»;
  • «MinIO (S3)»;
  • «Установка ELMA365 Standard» (раскомментировать параметр ELMA365_DB_S3_GATEWAY).

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Enterprise, читайте в статьях:

  • «MongoDB»;
  • «PostgreSQL»;
  • «MinIO (S3)»;
  • «Kubernetes кластер»;
  • «Подготовка встроенных баз данных» (установка RabbitMQ и Redis);
  • «Проксирование S3 в Kubernetes через S3-Gateway»;
  • «Проксирование баз данных в Kubernetes через DB‑Gateway»;
  • «Установка ELMA365 Enterprise в Kubernetes».
  1. Окружение без доступа в интернет.

Характеристики:

  • один физический сервер или виртуальная машина для ELMA365 On-Premises, компонентов проксирования и хранения данных;
  • три физических сервера или виртуальных машины для внешних баз данных;
  • один физический сервер или виртуальная машина для хранилища образов Kubernetes;
  • установка без доступа в интернет;
  • локальное хранилище образов Kubernetes;
  • встроенные компоненты хранения данных RabbitMQ, Redis;
  • внешние компоненты хранения данных MongoDB, PostgreSQL, S3 MinIO;
  • встроенный прокси-сервер HAProxy для проксирования входящих запросов к внешнему S3-хранилищу.

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Standard, читайте в статьях:

  • «Установка Harbor»;
  • «Загрузка образов ELMA365»;
  • «MongoDB»;
  • «PostgreSQL»;
  • «MinIO (S3)»;
  • «Установка ELMA365 Standard» (раскомментировать параметр ELMA365_DB_S3_GATEWAY).

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Enterprise, читайте в статьях:

  • «Установка Harbor»;
  • «Загрузка образов ELMA365»;
  • «Загрузка образов Deckhouse»;
  • «MongoDB»;
  • «PostgreSQL»;
  • «MinIO (S3)»;
  • «Kubernetes кластер»;
  • «Подготовка встроенных баз данных» (установка RabbitMQ и Redis);
  • «Проксирование S3 в Kubernetes через S3-Gateway»;
  • «Проксирование баз данных в Kubernetes через DB‑Gateway»;
  • «Установка ELMA365 Enterprise в Kubernetes».

Отказоустойчивая инфраструктура PROD-окружения

Этот вариант организации инфраструктуры подходит для окружений эксплуатации (PROD) с повышенными требованиями к надёжности компонентов хранения данных.

Характеристики:

  • 10 физических серверов или виртуальных машин для Kubernetes-кластера, в котором будут размещены ELMA365 On-Premises и компоненты хранения данных;
  • 10 физических серверов или виртуальных машин для внешних баз данных;
  • два физических сервера или две виртуальные машины для балансировки и проксирования входящих запросов к ELMA365 On-Premises, S3-хранилищу и Р7-Офис;
  • два физических сервера или две виртуальные машины для Р7-Офис;
  • один физический сервер или одна виртуальная машина для хранилища образов Kubernetes;
  • установка без доступа в интернет;
  • локальное хранилище образов Kubernetes;
  • встроенные отказоустойчивые компоненты хранения данных RabbitMQ, Redis;
  • внешние отказоустойчивые компоненты хранения данных MongoDB, PostgreSQL, S3 MinIO;
  • внешний отказоустойчивый сервер документов Р7-Офис;
  • внешний отказоустойчивый прокси-сервер HAProxy для проксирования входящих запросов к ELMA365 On-Premises, S3-хранилищу и Р7-Офис.

Подробнее о том, как развернуть этот вариант инфраструктуры для ELMA365 Enterprise, читайте в статьях:

  • «Установка Harbor»;
  • «Загрузка образов ELMA365»;
  • «Загрузка образов Deckhouse»;
  • «Кластер MongoDB»;
  • «Кластер PostgreSQL»;
  • «Кластер MinIO»;
  • «Отказоустойчивый Kubernetes кластер»;
  • «Подготовка встроенных баз данных» (установка RabbitMQ и Redis);
  • «Отказоустойчивый HAProxy»;
  • «Изменение параметров ELMA365 Enterprise»;
  • «Установка сервера документов Р7-Офис».
