Access to an app

Users who have access to an app can see the app in the left menu, interact with app items that are accessible to them, and use the app in search by parameters. Access can be configured when creating the app or later in its settings.

Access is granted to groups of users. Note that these groups also need to have access to the workspace that the app is stored in.

If you’re not included in a group that has access to an app, you won’t see it in the left menu, but you will be able to work with its items using direct links. Read more about access settings in Access permissions in BRIX.

начало внимание

Only the users included in the Administrators group can manage access to apps.

конец внимание

To configure an app’s access settings, click the gear icon next to the app’s name and select App Access.

In the window that appears, select the groups of users who should see the app in the workspace’s left menu.

Example 1

The CVs app should be available only to the HR department and department managers. To make this possible, in the app access settings, select HR department and Department Managers groups from the list, then save the changes.

Now users who are not included in the specified groups will not be able to see the app in the workspace’s left menu or access it using a direct link.

Example 2

An employee who was part of the HR department moved to another department. To restrict their access to the CVs app, delete them from the HR department group. The user won’t be able to see the app in the left menu anymore.

If access to the data in the CVs app is set up for the HR department group as well, the employee will loose access to app items of the app and won’t be able to open them using direct links.
