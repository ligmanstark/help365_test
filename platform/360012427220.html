E ELMA365 Store ELMA365 Store Каталог готовых решений для ELMA365.
T TypeScript TypeScript TypeScript SDK — это библиотека для работы с системой ELMA365 из пользовательских скриптов.
W Web API Web API Web API представляет собой инструмент, позволяющий получать данные из сторонних систем, обрабатывать их в ELMA365 и при необходимости передавать данные обратно в стороннюю систему. Например, при помощи Web API можно запускать бизнес-процессы из внешних приложений и получать нужные данные из этих процессов одним запросом.
Рус
А Администратор Administrator Пользователь ELMA365, обладающий возможностью создавать и изменять учётные записи пользователей, изменять настройки и назначать права доступа в системе.
А Ассоциированная лента Associated activity stream Лента сообщений, отображающаяся на карточке задач, файлов, элементов приложений. Помимо сообщений от пользователей, в ленте отображается история событий ассоциированного объекта.
Б Бизнес-процесс Business process Регулярно повторяющаяся последовательность взаимосвязанных действий, направленных на создание определённого продукта или услуги для потребителей. При этом потребителями могут быть не только внешние заказчики, но и внутренние.
Б Бизнес-процесс на уровне приложения Business process associated with an app Бизнес-процесс, который создаётся внутри приложения. При работе с таким процессом пользователь может настроить форму и статусы приложения, не покидая дизайнер процессов.
Б Бизнес-процесс на уровне раздела Business process associated with a workspace Бизнес-процессы на уровне раздела описывают логику его работы и могут связывать несколько приложений данного раздела. Например, для раздела Работа с клиентами можно добавить бизнес-процесс оформления заказа, в котором будут участвовать два приложения — Клиенты и Договор.
Б Бизнес-процесс, привязанный к приложению Business process associated with an app Бизнес-процесс, привязанный к кнопке Сохранить на карточке приложения. Как только пользователь добавляет новый элемент, привязанный процесс запускается с его данными.
В Вебхук Webhook Графический элемент Webhook используется для моделирования процесса. Позволяет автоматически передавать информацию об экземпляре процесса из ELMA365 во внешнюю систему.
В Версия бизнес-процесса Version of a business process Представляет собой один из вариантов бизнес-процесса. Версии различаются порядковым номером, статусом, датой создания, автором и описанием.
В Версия файла File version Вариант файла, в который пользователи внесли какие-либо изменения.
В Виджет Widget Элемент интерфейса или небольшое вспомогательное приложение с определённым функционалом для решения таких задач, как загрузка файла, отображение инструкций, ленты, списка пользователей, осуществления автоматических расчётов и т. д.
В Видимость приложения App visibility От настроек видимости зависит, какие пользователи увидят приложение в меню раздела. Приложение можно скрыть или сделать видимым для конкретных сотрудников, администраторов, групп пользователей.
В Видимость раздела Workspace visibility От настроек видимости зависит, какие пользователи увидят раздел в левом меню главной страницы. Раздел можно скрыть или сделать видимым для конкретных сотрудников, администраторов, групп пользователей.
В Воронка продаж Sales pipeline Принцип распределения сделок по стадиям процесса продаж.
Г Генерация по шаблону Generate from template Графический элемент Генерация по шаблону используется для моделирования схемы процесса. Он позволяет создавать документы с помощью загруженных в ELMA365 шаблонов. Генерация осуществляется во время исполнения процесса, при этом данные из контекста процесса автоматически подставляются в сгенерированный документ.
Г Главная страница Main page Главная страница ELMA365, которая отображается сразу после авторизации пользователя в системе или при нажатии на кнопку главного меню.
Г Группа Group Элемент организационной структуры, представляющий собой группу пользователей с общими должностными обязанностями.
Г Группа пользователей User group Группы используются для настройки доступа к файлам, документам, приложениям, элементам приложений, разделам, а также для определения исполнителей в зонах ответственности в бизнес-процессах.
Г Групповой чат Group chat Чат, в котором участвуют не менее трёх пользователей. Позволяет синхронизировать работу внутри команды, оставаться в курсе последних событий, обмениваться файлами и быстро решать рабочие вопросы.
Д Дизайнер бизнес-процессов Process designer Раздел системы, предназначенный для моделирования бизнес-процессов.
Д Динамика сделок Deal Dynamics Инструмент для получения информации о текущем состоянии сделок по выбранной воронке продаж. С помощью него можно отслеживать статусы сделок за определённый период.
Д Динамические формы Dynamic forms Формы задачи или страницы, внешний вид которых может меняться в зависимости от данных, внесённых пользователем в систему в процессе работы. Это позволяет не перегружать формы полями и кнопками, а показывать их только тогда, когда необходимо.
Д Документ Document Приложение типа Документ, предназначенное для создания документов в ELMA365, например, договоров, заявлений, счетов на оплату и т. д.
Д Должность Job position Элемент организационной структуры, обозначающий одного сотрудника компании. Примеры должностей: директор, главный бухгалтер, секретарь и т. д.
Д Дополнительные параметры Additional parameters Параметры для написания скриптов внутри бизнес-процессов. В отличие от контекстных переменных, значение которых меняется от экземпляра к экземпляру, параметр остаётся неизменным для всех экземпляров процесса.
Ж Жизненный цикл документа Document lifecycle Все этапы (статусы), которые проходит документ в компании от создания до отправки в архив или уничтожения.
З Задача Task Один из инструментов для организации работы внутри компании. Сотрудники устанавливают сроки и отслеживают выполнение задач, обмениваются нужной информацией и документами — всё для того, чтобы получить желаемый результат вовремя.
З Задача в процессе Process task Задача, назначаемая в рамках бизнес-процесса. Выполнение задачи по процессу обеспечивает завершение экземпляра процесса и достижение необходимого результата.
З Закрытие согласования End Approval Графический элемент Закрыть согласование используется как вспомогательный для элемента Согласование при построении маршрутов движения документов. Он размещается на том отрезке, где необходимо завершить согласование документа и вынести резолюцию, не дожидаясь окончания бизнес-процесса.
З Замещение Substitution Временное исполнение обязанностей отсутствующего сотрудника другим сотрудником. В ELMA365 существует три типа замещения: информирование, переназначение задач и полная передача прав. Они отличаются набором передаваемых прав.
З Запуск процесса Start Process При моделировании элемент Запуск процесса используется для декомпозиции и повышения наглядности схем. Он позволяет вызывать подпроцесс в рамках родительского процесса. Подпроцесс по своей сути — это самостоятельный бизнес-процесс, функционал которого является частью более крупного родительского процесса.
З Зона ответственности Swimlane Определяет исполнителя задач, размещённых в зоне ответственности.
И Исключающий ИЛИ-шлюз Exclusive Gateway Активирует работу процесса только для одного из исходящих переходов. Для данного шлюза обязательно должен быть задан переход по умолчанию.
И Исполнительская дисциплина Schedule compliance Соотношение количества задач, выполненных вовремя, к общему количеству поставленных задач. Она рассчитывается в процентах и включает задачи, закрытые на текущий момент с начала месяца.
К Календарь личный Personal calendar Раздел системы, позволяющий планировать рабочую деятельность — создавать события и встречи, просматривать календари подчинённых. В личном календаре также отображаются события из календарей других разделов, с которыми текущий пользователь как-то связан.
К Календарь раздела Workspace calendar Календарь, созданный при помощи приложения типа Событие внутри определённого раздела.
К Канал Channel Удобный инструмент для обмена информацией внутри команд и отделов. С помощью каналов можно синхронизировать работу сотрудников, делиться новостями компании, предупредить коллег о командировке или отпуске, поздравить с важным событием, рассказать о новых сотрудниках.
К Канбан-доска Kanban board Наглядный инструмент для отслеживания текущего состояния бизнес-процессов. Каждый столбец доски соответствует определённому этапу работ. По мере выполнения бизнес-процесса, элементы автоматически перемещаются по доске слева направо, демонстрируя прогресс исполнения.
К Карточка задачи Task page Страница, открывающаяся при нажатии на название задачи. На ней отображается вся необходимая информация для выполнения работы: описание задачи, вложения, сроки, участники, автор и процент выполнения.
К Карточка экземпляра Instance summary page Страница, открывающаяся при нажатии на название процесса в разделе Задачи > Исходящие. Здесь отображается: дата старта и завершения, версия процесса, имя инициатора и карта процесса, демонстрирующая ход выполнения задач в реальном времени. При необходимости вы можете прервать выполнение экземпляра с его карточки.
К Карта процесса Process map Графическое представление процесса, которое показывает ход выполнения экземпляра процесса и текущие операции по нему.
К Карточка элемента приложения App item page Страница, открывающаяся при создании, просмотре или редактировании элемента приложения. На карточках имеются поля, предназначенные для заполнения, например, название, контактное лицо, номер телефона и т. д. Кроме того, на форму можно добавить таблицы, списки и переключатели, которые позволяют пользователю выбрать ту или иную опцию.
К Ключевая переменная Key variable Переменная, участвующая в автоматическом формировании наименования элемента приложения.
К Компания Company Раздел ELMA365, в котором на отдельных страницах собрана основная информация о вашей организации: структура, сотрудники, график отсутствий, замещения, исполнительская дисциплина и т. д.
К Компонент ELMA365 Store ELMA Store component Готовое решение (пакет разделов, приложений и процессов), предназначенное для расширения базовых возможностей ELMA365.
К Конечное событие End Event Осуществляет завершение экземпляра процесса. При его достижении процесс считается завершённым.
К Конструктор форм Form builder Инструмент, который позволяет проектировать и изменять формы создания, редактирования и просмотра приложений, задач и событий.
К Контекст бизнес-процесса Business process context Все данные, которые создаются, изменяются и сохраняются во время выполнения конкретного процесса. Например, информация об участниках, файлы, комментарии исполнителей, контрагенты, формы и т. д. Эти данные хранятся в переменных, значения которых задаются либо самими сотрудниками в ходе выполнения процесса, либо определяются системой автоматически.
К Контекстная переменная Context variable Величина, изменяющая свое значение от одного экземпляра процесса с другому. Например, переменная Инициатор хранит информацию о конкретном пользователе, запустившем экземпляр бизнес-процесса. Значение такой переменной будет уникальным для каждого нового экземпляра.
К Контрагент Contractor Компания, с которой сотрудничает предприятие.
Л Лента сообщений Activity stream В #ленте отображаются все последние новости из информационных каналов, участником которых вы являетесь, и системные оповещения. В #ленте вы можете просматривать сообщения, комментировать новости, делиться файлами и создавать задачи коллегам.
М Маршрут документа Document route Путь перемещения документа в процессе его обработки. У каждого этапа маршрута есть свой исполнитель, конкретные задачи и сроки.
М Множественное исполнение Multiple execution Настройка, позволяющая назначить задачу одновременно всем сотрудникам, обозначенным в качестве исполнителей зоны ответственности.
М Монитор процессов Process monitor Аналитический инструмент для системных администраторов, который позволяет отслеживать состояние процессов в реальном времени.
Н Неисключающий ИЛИ-шлюз Inclusive OR-gateway Активирует работу процесса всех исходящих переходов, для которых выполняется условие перехода.
Н Нумераторы Counters Набор правил, которые используются для генерации последовательных номеров для элементов приложения. Например, для присвоения номера документу при регистрации.
О Ознакомление Send Document/App Item Графический элемент Ознакомление используется для создания схемы бизнес-процесса. Он размещается на том этапе, где участники процесса должны ознакомиться с содержимым файла, документа или элемента приложения. В #ленту сотрудников приходит сообщение, содержащее ссылку на объект, а на карточке объекта создается лист ознакомления.
О Оповещение Notifications Графический элемент Оповещение используется при моделировании бизнес-процесса и располагается на тех участках, где пользователей необходимо проинформировать о важных событиях, например о согласовании или отклонении документа, оплате счета и т. д.
О Организационная структура Organizational Chart Представляет структуру организации в виде схемы, которая отражает всю иерархию предприятия (должность, отдел, подразделение).
О Отдел Department Элемент организационной структуры, обозначающий один отдел компании и позволяющий создавать внутри себя группы и отдельные должности.
О Отладка процесса Process debugging Этап разработки бизнес-процесса, в ходе которого обнаруживают и исправляют ошибки. Во время отладки процесс воспроизводится пошагово в режиме реального времени. Таким образом вы можете проверить логику операций, оценить, насколько удобно и понятно настроены формы задач, убедиться, что все скрипты отрабатывают верно.
О Отправка на согласование Send for approval Встроенный механизм, позволяющий отправить файл или элемент приложения по стандартному маршруту согласования одному или нескольким пользователям. Поддерживается как параллельное, так и последовательное согласование. Список сотрудников, рассмотревших файл или элемент приложения, и их резолюции отображаются в листе согласования.
О Отправка на ознакомление Send for information Встроенный механизм, позволяющий отправить файл или элемент приложения на ознакомление одному или нескольким пользователям. Кроме того, настройки позволяют создавать лист ознакомления, в котором фиксируется информация о всех сотрудниках, прочитавших документ.
О Отчёт Report Выборка данных, систематизированных по определённым критериям и предназначенных для проведения анализа деятельности пользователей или компании в целом.
П Папка Folder Папки используются для хранения и упорядочивания файлов. Вы можете создавать и редактировать новые папки, а также управлять правами доступа к ним.
П Параллельный шлюз Parallel Gateway Активирует работу процесса для всех исходящих переходов. Необходим для запуска процесса по нескольким маршрутам одновременно.
П Переход Connector Графический элемент, использующийся для соединения задач, событий, шлюзов и других операций на схеме процесса. Переход задает порядок выполнения действий и определяет последовательность событий в процессе. Подписи к переходам повышают наглядность схемы, а также служат названиями кнопок на формах задач.
П Приложение App Компонент ELMA365, предназначенный для создания и хранения элементов приложения и бизнес-процессов. Например, приложение Компании используется для хранения данных о конкретных юридических лицах, их адресах, телефонах и т. д. Каждая конкретная компания, добавленная в приложение Компании, и будет являться его элементом.
П Приложение типа «Документ» Document type app Приложение типа Документ предназначено для создания документов в ELMA365, например, договоров, заявлений, счетов на оплату и т. д.
П Приложение типа «Событие» Event type app С помощью приложений данного типа вы можете создавать календари в различных разделах системы.
П Пользовательская форма Custom form Форма, спроектированная и настроенная с помощью конструктора форм для определённых потребностей пользователя или компании.
П Присваивание Assign Value Графический элемент Присваивание используется для того, чтобы автоматически передавать данные в бизнес-процесс и устанавливать соответствие между полями приложения и контекстным переменным.
П Прогноз поступлений Revenue Forecast Отчёт о запланированных денежных поступлениях: сумме, статусе, планируемой дате, и т. д.
П Промежуточное событие-таймер Timer intermediate event Приостанавливает процесс на определённое время. Количество промежуточных событий не ограничено, они могут располагаться на любом отрезке между начальным и конечным событиями.
П Публикация процесса Process publishing Чтобы процесс стал исполняемым, его необходимо опубликовать. После этого ход процесса будет выполняться автоматически: исполнителям назначаться задачи, отправятся уведомления, произойдёт приостановка процесса в соответствии с его схемой.
Р Раздел Workspace Раздел объединяет одно или несколько приложений, связанных общей тематикой. Например, внутри раздела Документы могут быть собраны такие приложения, как Договор, Приказ, Служебная записка, Акт. Функция разделов, в первую очередь, организационная.
Р Разделитель Separator Разделитель предназначен для группировки приложений внутри раздела. Он визуально отделяет приложения друг от друга разделительной чертой.
Р Расширенный поиск Advanced search Функция, позволяющая осуществлять поиск объектов системы по набору параметров.
Р Расширенный режим настройки форм Advanced Mode Расширенный режим настройки форм позволяет создавать и настраивать пользовательские формы для карточек создания, просмотра и редактирования элемента приложения. В частности, вы сможете расположить поля форм в нужной последовательности, изменить их внешний вид, указать, какие поля обязательны для заполнения, а какие доступны только для чтения. Кроме того, в расширенном режиме можно создать шаблон форм, а также настроить отображение элементов приложения.
Р Регистрация Registration Графический элемент Регистрация помогает организовать учёт и контроль документов в рамках компании путём присвоения регистрационного номера (индекса) документу.
Р Родительская задача Parent task Задача, которая декомпозируется на подзадачи, т. е. со страницы которой создаются подзадачи.
Р Роль Role На роль можно назначить только одного пользователя или должность из организационной структуры. Роли используются для настройки доступа к файлам, документам, приложениям, элементам приложений, разделам, а также для определения исполнителей в зонах ответственности бизнес-процессов.
С Сделка Deal Объект ELMA365, хранящий информацию о соглашении, которое компания заключила с определённым контрагентом.
С Скрипт Script Код на языке TypeScript, который позволяет обращаться к данным бизнес-процессов, создавать, запрашивать, изменять и удалять элементы приложений, а также взаимодействовать с внешними системами по протоколу HTTP.
С Событие Event События влияют на ход процесса или на время выполнения его отдельных этапов. Они могут запускать процесс, приостанавливать его исполнение, завершать определенную ветку. На схеме события изображаются в виде окружности. Существует три основных типа событий: Стартовое, Промежуточное и Конечное.
С Событие календаря Calendar event Запись о некотором непериодическом мероприятии.
С Согласование Approval Графический элемент Согласование используется для построения маршрутов движения документов. Он размещается на том отрезке процесса, где сотрудник, например, руководитель, должен утвердить или отклонить документ.
С Страница Page Компонент системы, на котором можно разместить виджеты для отображения нужной информации. Вы можете создать страницу как внутри раздела, так и в левом меню главной страницы.
С Ссылка Link Ссылка добавляется в левое меню страницы или раздела и позволяет быстро перейти к стороннему интернет-ресурсу или открыть веб-документ.
С Ссылка на папку Link to a folder Ссылки на папки помогают систематизировать информацию и быстро находить нужные или часто используемые папки. Ссылку можно создать в любой папке.
С Стартовое событие Start Event Инициирует запуск процесса и указывает, в какой точке он берет начало. Процесс может содержать только одно Стартовое событие.
С Статус приложения App status Статусы предназначены для отслеживания текущего состояния элемента приложения.
С Статус события Event status Повышает наглядность календаря и позволяет быстро ориентироваться в запланированных мероприятиях. При наличии большого числа событий, их можно сортировать, например, по статусу Открыто, Отменено, Закрыто, что позволяет легко контролировать расписание.
С Супервизор Supervisor Системная роль, на которую по умолчанию назначается пользователь, создавший компанию в ELMA365. Супервизор обладает полным набором прав и получает задачи по бизнес-процессу в том случае, если исполнителя не удалось определить автоматически.
С Схема процесса Flow chart Визуальная схема, на которой наглядно изображён весь процесс: участники, задачи, события, операции и переходы между ними. Схемы процессов, смоделированные в ELMA365, являются исполняемыми. Это значит, что все задачи, уведомления и т. д. будут автоматически приходить соответствующим сотрудникам на нужном этапе после публикации схемы.
Т Таблицы Tables Особый тип поля приложения. Добавив его на форму приложения, вы сможете структурировать данные по строкам и колонкам, изменять их внешний вид и вставлять специальные формулы, которые будут выполнять расчеты.
Т Тип данных/Тип полей в системе Data type/Field type Характеристика набора данных, определяющая диапазон его возможных значений, список допустимых операций, а также способ хранения данного набора данных.
Т Токен Token Токен используется для авторизации запросов к API. Например, чтобы получить элементы приложения или запустить бизнес-процесс из внешней системы с помощью API.
У Упоминания Mentions Удобный способ привлечь внимание коллег к файлу, задаче или комментарию в чате или канале. Если вы упоминаете человека с помощью символа @, он получает всплывающее уведомление о вашем сообщении.
У Управление статусом Manage Status Графический элемент Управление статусом применяется только в тех бизнес-процессах, в которых используются приложения с настроенными статусами. С его помощью этого графического элемента можно автоматически изменять статус приложения на том или ином этапе процесса и таким образом контролировать выполнение работ.
У Участник процесса Process participant Исполнитель одной из зон ответственности.
С Участник задачи Task participant Пользователь, участвующий в работе над поставленной задачей, но не являющийся ответственным за её выполнение.
У Учетная запись User account Логин и пароль пользователя, необходимые для входа в систему ELMA365 и предоставления доступа к личным данным и настройкам.
Ф Файл File Документы форматов .docx, .xlsx, .pdf, изображения, аудио и видео, загруженные в ELMA365 с компьютера пользователя или созданные непосредственно в системе.
Ф Фильтр Filter Сохранённый набор параметров расширенного поиска объектов.
Ф Форма элемента приложения/задачи/запуска процесса App item/task/process start form Под формой понимается внешний вид карточки элемента приложения или задачи, а также окна запуска процесса. При настройке формы указывается, какие данные на ней будут отображаться и какие будут доступны для редактирования.
Ч Чат Chat Чаты используются для личного взаимодействия между сотрудниками, своевременного обмена важными данными, координации деятельности, быстрого решения вопросов.
Ч Чек-лист Check list Перечень действий, которые необходимо выполнить. Чек-лист позволяет разбить крупные задачи на этапы и пошагово планировать работу. Чек-листы можно создавать для задач, файлов и элементов приложения.
Ш Шаблон документа Document template Файл в формате .docx или .xlsx, содержащий определённый набор переменных. Значения переменных автоматически заполняются из данных, хранящихся в контекст бизнес-процесса.
Ш Шаблон формы Form template Шаблоны формы используются в тех случаях, когда функционала стандартной формы недостаточно. Шаблон позволяет добавить новые кнопки, виджеты, менять расположение полей и т. д. Вы можете создавать шаблоны форм для задач, событий, карточек экземпляров процесса и приложений.
Ш Шлюз Gateway Шлюзы представляют собой точки принятия решения в процессе. Они используются для того, чтобы направить процесс по той или иной ветке, в зависимости от определённых условий.
Э Экземпляр бизнес-процесса Business process instance Экземпляр создается каждый раз, когда кто-то запускает опубликованный процесс в ELMA365.
Э Экспорт данных приложения App data export Выгрузка хранящихся в приложении данных в виде файла формата .xlsx или .csv.
Э Экспорт приложения App export Выгрузка приложения в файл формата .e365. При экспорте приложения вместе с ним выгружаются: бизнес-процессы, настроенные статусы и фильтры, нумераторы, группы и роли, формы и контекстные переменные, используемые в бизнес-процессах, настройки кнопок, страницы и разделители.
Э Экспорт раздела Workspace export Выгрузка раздела со всеми приложениями.
Э Элемент организационной структуры Org chart item Составная часть графической модели организационной структуры (отдел, должность, группа).
Э Элемент приложения App item Карточка на странице приложения, которая отображает определённую информацию. Элемент создаётся пользователями и хранит в себе внесённые ими данные.
