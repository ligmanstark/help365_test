My tasks

This widget is used on pages to display a list of a user's tasks.

To configure the widget, drag it to the canvas or click +Widget. Enter the settings in the opened window.

Main tab

  • Show header. You can hide or show the Tasks workspace title in the widget header.
  • Show the Active/All switch. Here you can show or hide the switch between the Active and All task lists. If hidden, the user will only be able to see the current tasks assigned to them. The completed tasks will be hidden from the lists, the user will not be able to view their pages. This is done if you need to limit access to information about routes and process administration that are available to the pages of completed tasks.
  • Show buttons in header. You can hide or show the +Process and +Task buttons in the widget header.

You can also add custom buttons to display in the widget header. Click +Button in the upper right corner of the canvas and configure a button. Read more in the Quick action buttons article.

You can set system option values for the widget’s main settings. Click the icon next to the field's name and then the <Not defined> link. Select an available system field: Hide, Read only, Load the contents asynchronously.

Events and System tabs

The Events and System tabs are for system settings that are the same for all the widgets. They allow you to configure a widget’s visibility and access to its data, define what happens, when the user moves or hovers the pointer over the widget, etc. Find out more in System widget settings.

To complete the settings, click Save. To make the changes available to users, click Save and Publish in the top toolbar.

Use case

In the case below, the My tasks widget is configured on the same page as the activity stream.

Additionally, you can use this widget to set up external portal pages. In such case the Active/All switch and the buttons for adding tasks and processes remain hidden all the time.
