Системные справочники

В главном меню ELMA365 по умолчанию настроен системный раздел Системные справочники.

Системный справочник — это набор данных, которые нужны для работы в разных разделах системы. Для удобства эти данные вынесены в отдельные системные приложения:

  • Мои юридические лица — данный справочник предназначен для хранения реквизитов юридических лиц вашей организации и удобен для оформления договоров, соглашений, сделок и так далее;
  • Внешние пользователи — в этот справочник добавляются пользователи, которым вы хотите предоставить доступ к работе на внешнем портале любого раздела. Подробнее об этом мы написали в статье «Внешние пользователи»;
  • График обслуживания — в этом справочнике задаются рабочие графики для операторов из раздела Линии и ServiceDesk 2.0. Здесь можно указать параметры для рабочих и выходных дней, а также добавить исключения в виде укороченных или праздничных дней. Графики обслуживания используются при настройке SLA для линий. Данные из этого справочника нельзя выгружать из-за особенностей их формата.

Для расширенной работы с исключениями вы можете обновить приложение График обслуживания. Например, чтобы использовать строку поиска и фильтровать записи. Для этого импортируйте в систему файл формата .е365.

  • Сотрудники — в этом справочнике можно хранить дополнительную информацию о пользователях системы. Например, логин сотрудника в Skype, его режим работы, номер кабинета и т. д. Для этого добавьте на форму нужные поля. Вы можете добавить сотрудника в справочник вручную, если он является пользователем системы со статусом Активный. Также элемент в справочнике создаётся автоматически, если:
    • пригласить пользователя в разделе Администрирование > Пользователи;
    • импортировать пользователя из AD/LDAP, и при этом в настройках модуля включена опция Автосоздание сотрудника;
    • пользователь авторизуется через OAuth2 или SAML, и в этих модулях настроено создание пользователя ELMA365.

Данные пользователей, импортированных из AD/LDAP, можно записывать в справочник автоматически при синхронизации с сервером. Для этого в настройках модуля нужно включить опцию Обновление сотрудника и сопоставить поля приложения Сотрудники с атрибутами AD/LDAP. Подробнее читайте в статье «Запись информации о сотрудниках из AD/LDAP».

Начало примечание 

Примечание

Справочник Сотрудники следует отличать от страницы в разделе Компания > Сотрудники, которая носит информационный характер и позволяет ставить задачи и отправлять сообщения коллегам. Дополнительные данные пользователей в справочнике Сотрудники можно использовать при разработке бизнес‑процессов и решений, в других разделах системы, а также экспортировать и импортировать, как данные любого системного приложения.

Конец примечание 

Чтобы добавить новый элемент, на странице справочника в правом верхнем углу нажмите кнопку, например, + Юридическое лицо. Затем заполните поля на карточке элемента и сохраните данные.

Элементы справочника и хранящуюся в них информацию можно использовать в других приложениях и бизнес-процессах.

Набор свойств в системных справочниках определён заранее. Вы можете создать новые поля, например, дополнить форму справочника Мои юридические лица в соответствии с реквизитами вашей компании.

Вы также можете загрузить или выгрузить данные из системных справочников. Для этого нажмите значок в правом верхнем углу страницы и выберите нужное действие. Эта опция недоступна в системном справочнике График обслуживания.

Обратите внимание, действия с системными справочниками, как и с любыми системными приложениями, имеют ряд особенностей и ограничений, в том числе при их экспорте.

Для раздела Системные справочники доступны стандартные настройки: вы можете добавить бизнес‑процессы, шаблоны документов, настроить группы пользователей и т. д. Чтобы открыть меню настроек, нажмите на значок шестерёнки рядом с названием раздела.
