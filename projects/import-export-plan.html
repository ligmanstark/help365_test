Импорт и экспорт плана

Если вы работаете с проектами во внешних системах, например в MS Project, используйте возможность импорта и экспорта плана проекта. Также вы можете переносить план с одной площадки ELMA365 на другую.

Импорт плана

Вы можете импортировать план из файла формата .mpp или .xml, добавив его задачи в уже существующую версию плана. В результате импорта создаётся новая версия плана. В ней сохранятся все задачи исходной версии. К ним добавятся импортированные задачи с учётом их вложенности.

Для этого:

  1. В карточке проекта перейдите на вкладку Календарный план и нажмите кнопку Редактировать.
  2. На панели инструментов в строке поиска выберите версию плана, в которую хотите импортировать задачи.
  3. Нажмите кнопку .
  4. В открывшемся окне загрузите файл формата .mpp или .xml и установите настройки:
    • Импортировать завершенные задачи — вы можете добавить к текущему плану только активные задачи плана из внешней системы. Для этого снимите флажок в этой опции;
    • Объединить планы — включите опцию, если вы импортируете план, который ранее экспортировали из системы. Это позволит не дублировать задачи с одинаковыми ID, существующая задача заменится на импортированную. При этом сравниваются ID задач в ELMA365;
    • Установить исполнителя элементам без исполнителя — выберите ответственного за импортированные задачи с исполнителями, Ф. И. О. которых не найдены среди пользователей ELMA365. Все эти задачи будут назначены выбранному исполнителю после завершения импорта. Если оставить поле незаполненным, вы можете выбрать ответственных для таких задач во время импорта.

  1. Нажмите Импорт.

Когда процесс импорта завершится, вы увидите новую версию плана, в которой к созданным ранее задачам добавятся импортированные. Такая версия плана сохраняется автоматически.

В импортированных задачах:

  • сохраняется их вложенность;
  • поля Название, Ответственный, Дата начала* и Дата окончания* заполняются данными из внешней системы;
  • если исходный тип задачи не поддерживается системой, полю Тип* присваивается значение Задача, а связанному с ним полю Контроль* — Оповестить при выполнении;
  • всем исполнителям автоматически назначаются права Участника проекта.

Если план переносится с одной площадки ELMA365 на другую, то в импорте также участвуют пользовательские поля задач с типами Строка, Число, Выбор «да/нет», Дата/время, Категория, Деньги, Учетная запись, Номер телефона, Электронная почта, Ф. И. О., Ссылка, Роль, Пользователи. В импортированных задачах поля указанных типов заполнятся данными с внешней площадки ELMA365. При этом важно, чтобы поля и их типы совпадали на обеих площадках системы. Обратите внимание, пользовательские поля с типами Приложение, Произвольное приложение, Файлы и Изображение нельзя импортировать.

После импорта плана:

  1. Если в добавленных задачах есть ответственные, которым нужно настроить права доступа, отличные от прав Участника проекта, обратитесь к администратору.
  2. В добавленной задаче с типом Процесс вручную настройте входные параметры импортированного процесса. Для этого откройте форму задачи и внесите данные в блоке Бизнес-процесс.

Конфликты исполнителей

Если исполнители импортируемых задач не являются пользователями ELMA365 и вы не назначили ответственного для таких задач, во время импорта откроется окно, в котором нужно установить исполнителей. Вы можете:

  • выбрать ответственным любого пользователя, нажав значок лупы;
  • назначить задачи на себя. Для этого нажмите кнопку мне;
  • назначить задачи руководителю проекта. Для этого нажмите кнопку руководителю проекта.

Когда сопоставите исполнителей для каждой задачи, нажмите Импорт.

Экспорт плана

Вы можете выгрузить план в файл формата .mpp или .xml. Для этого:

  1. В карточке проекта перейдите на вкладку Календарный план и нажмите кнопку Редактировать.
  2. На панели инструментов в строке поиска выберите версию плана для экспорта.
  3. Нажмите кнопку и выберите .mpp или .xml.

Если в плане проекта есть пользовательские поля, то их значения можно перенести на другую площадку ELMA365. В экспорте участвуют такие же типы данных, как и при импорте плана.

После этого файл с планом проекта выгрузится на ваш ПК.

publish-project-plan.html project-tasks.html
