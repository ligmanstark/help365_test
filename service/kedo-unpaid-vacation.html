Отпуск без сохранения заработной платы

Сотрудник может создать заявку на внеплановый отпуск без сохранения заработной платы. В такой заявке нужно выбрать основание из доступных в справочнике Причины на отпуск без сохранения ЗП раздела УРВ. Это могут быть причины, предусмотренные законодательством, документами компании и прочие. В соответствии с ТК РФ для элементов справочника заданы периодичность, максимальная длительность отпуска, а также необходимость приложить подтверждающие документы.

Если заявка создана, запустится процесс оформления отпуска.

Цель процесса

Согласовать отпуск руководителем сотрудника, оформить приказ на отпуск.

Результат процесса
  • подписанный приказ на отпуск по организации, в которой трудоустроен сотрудник;
  • по решению руководителя может быть запущен процесс совмещения на период отсутствия сотрудника

Участники процесса
  • сотрудник, который оформляет заявку на отпуск;
  • руководитель сотрудника;
  • сотрудник отдела кадров;
  • пользователь 1С;
  • подписант.

Подробнее читайте в статье «Оформление отпусков».

 

Этапы оформления отпуска без сохранения заработной платы:

  1. Создание сотрудником заявки на отпуск на портале КЭДО.
  2. Согласование отпуска руководителем сотрудника.
  3. Подготовка и подписание приказа на отпуск.

Создание заявки на отпуск без сохранения заработной платы

На портале КЭДО любой сотрудник может сформировать заявку на отпуск без сохранения заработной платы. Для этого в каталоге услуг в папке Отсутствия он выбирает вариант Отпуск без сохранения заработной платы.

В заявке на отпуск сотрудник:

  • выбирает причину отпуска из элементов справочника Причины на отпуск без сохранения ЗП;
  • указывает даты начала и окончания отпуска. При этом длительность отпуска можно выбрать только в рамках максимальной, если она задана для конкретной причины;
  • прикладывает документы, если они требуются для выбранной причины отпуска.

Если заявка создана, запустится процесс оформления отпуска. Заявка поступит на согласование руководителю сотрудника.

Движение и статус созданной заявки можно отслеживать на портале КЭДО разделе Заявки.

Согласование отпуска без сохранения заработной платы

После создания заявки на отпуск без сохранения заработной платы руководитель сотрудника получает задачу согласовать её. Если руководитель является внутренним пользователем, он может работать с задачей в интерфейсе ELMA365 и на портале КЭДО в разделе Задачи. Для внешнего пользователя задача доступна на портале КЭДО.

В карточке задачи:

  • доступна информация о сотруднике, его должность, а также данные заявки, созданной сотрудником:
    • причина отпуска;
    • даты и период отпуска;
    • приложенные документы, если они требуются для выбранного варианта причины;
  • руководитель может отметить необходимость замещения сотрудника на период отпуска.

Согласованная заявка поступит в отдел кадров для подготовки приказа на отпуск. Если требуется замещение, параллельно запустится процесс совмещения должностей и руководитель сотрудника получит задачу создать заявку на совмещение.

Если заявка на отпуск отклонена, инициатор процесса может откорректировать её по комментарию от руководителя и повторно отправить на согласование.

Для руководителя информация о статусе заявки его сотрудника доступна на портале КЭДО в разделе Заявки.

Подготовка и подписание приказа на отпуск без сохранения заработной платы

Ход процесса после согласования заявки зависит от того, настроена настроена в ELMA365 интеграция с 1С для кадровых процессов или нет.

  1. Если интеграция с 1С не используется, сотруднику отдела кадров поступает задача внести данные для приказа на отпуск. В ней доступны вся исходная информация о сотруднике и его заявка на отпуск. После проверки заявки сотрудник отдела кадров:
    • указывает в задаче номер и дату приказа для формирования файла приказа по шаблону;
    • если заявка некорректна — отправляет её на доработку инициатору процесса с указанием причины.
  1. Если интеграция с 1С настроена, данные заявки на отпуск передаются в 1С. На их основе ответственный пользователь в 1С выполняет проверку исходных данных. После этого ответственный пользователь:
    • формирует в 1С приказ на отпуск и отправляет его в ELMA365 для формирования файла приказа по шаблону;
    • если исходные данные некорректны — отправляет заявку на доработку. Сотруднику отдела кадров поступает задача проверить заявку. После проверки заявка поступает на доработку инициатору процесса.

После подготовки данных для приказа:

  1. Формируется файл приказа на отпуск по шаблону.
  2. Подписант со стороны организации получает задачу подписать приказ на отпуск. В задаче можно просмотреть файл приказа и заявку инициатора.
  3. После подписания приказа со стороны работодателя аналогичная задача поступает сотруднику — инициатору процесса. После выполнения задачи процесс завершается.
