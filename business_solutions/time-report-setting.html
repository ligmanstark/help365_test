Трудозатраты / Первичная настройка решения

Первичная настройка решения

  1. Импортируйте готовое решение из каталога ELMA365 Store или файла формата .e365. Модуль и раздел решения устанавливаются в процессе импорта.
  2. Настройте группы и роли пользователей в разделе.

Роли и группы используются:

  • для ограничения доступа к разделу, страницам и справочникам;
  • для определения участников бизнес-процессов.

В разделе Трудозатраты создана группа пользователей Согласующие. Добавьте в неё сотрудников, которые имеют право согласовывать трудозатраты, например, руководителей всех подразделений. Для группы Согласующие также доступен поиск трудозатрат по любому пользователю.

  1. Настройте права доступа и видимость страниц для пользователей. По умолчанию заданы следующие настройки доступа:
  • заполнять отчёт о трудозатратах, отправлять их на согласование, осуществлять поиск трудозатрат могут все пользователи;
  • согласовывать трудозатраты и осуществлять их поиск по любому автору трудозатрат могут пользователи, включённые в группу Согласующие;
  • все страницы раздела видны всем пользователям;
  • справочники Записи о трудозатратах и Виды деятельности отображаются только для администраторов системы. О назначении прав доступа к справочникам читайте в статьях «Доступ к приложению» и «Доступ к данным приложения».
  1. Перейдите в раздел Администрирование > Модули > Трудозатраты, настройте работу пользователей с трудозатратами и нажмите Сохранить.

  • Разрешить подачу трудозатрат по задачам — включите опцию, чтобы пользователи могли подавать трудозатраты по любым задачам, кроме проектных.
  • Разрешить подачу трудозатрат по проектам — включите опцию, чтобы пользователи могли работать с трудозатратами в проектах.
  • Разрешить подачу трудозатрат по приложениям — включите опцию, чтобы пользователи могли подавать трудозатраты из карточки элемента приложения, например из карточки сделки. Затем нажмите + Добавить, укажите название приложения, namespace и code. Используйте ссылку на приложение. Например, ссылка на приложение Сделки имеет вид https://хххххххххelma365.ru/_clients/_leads. В поле Namespace введите _clients, а в поле Code  укажите _leads. Чтобы удалить приложение из списка, нажмите значок корзины.
  • Максимальное время трудозатрат за день, час — укажите объём трудозатрат в часах, который можно подать за один день.
  • Максимальное время для подачи задним числом, дней — укажите количество календарных дней, за которое можно подать трудозатраты задним числом. В этом периоде учитывается текущий день. Например, чтобы установить срок не более одной недели, укажите в этом поле 8 дней.
  1. Заполните справочник Виды деятельности. Это позволит группировать трудозатраты для их учёта и анализа, а также устанавливать в проектах лимиты по видам работ. В карточке каждого вида деятельности укажите его название, например, Внутренние работы, Работы по внешнему проекту, Обучение, Продажи. Заполняя запись о трудозатратах, пользователь выберет элемент справочника.
  2. Для работы с трудозатратами в проектах, настройте карточку проекта и карточку проектной задачи, добавив на их формы виджет Трудозатраты.

Первичная настройка решения завершена.

Настройки справочника «Записи о трудозатратах»

В этом справочнике хранятся все созданные пользователями трудозатраты. Работать с ним по умолчанию может только администратор.

Справочник является приложением, поэтому в нём доступны стандартные настройки любого приложения. Вы можете создавать, редактировать и удалять записи, изменять их статусы, импортировать и экспортировать данные, предоставлять доступ к справочнику.

Чтобы открыть меню настроек, нажмите на значок шестерёнки рядом с названием справочника.

Для удобства в этом приложении настроен иерархический справочник, трудозатраты группируются в папках по статусам: На согласовании, Согласовано, Отказано.
