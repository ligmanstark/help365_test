Создание доверенности с помощью JSON

Если вы настроили пользовательский процесс с подпроцессом создания доверенности, вы можете формировать доверенность автоматически с помощью JSON.

В этом случае:

  • сотруднику не нужно заполнять данные для создания доверенности вручную;
  • доверенность не отправляется на подписание и регистрацию. Она отображается в разделе МДЧ > Доверенности, где можно собрать несколько доверенностей и отправить их вручную.  

При формировании доверенности с помощью JSON используются только существующие записи в справочниках Юридические лица, Физические лица, Индивидуальные предприниматели и Представитель. Новые записи не создаются.

Чтобы настроить обмен данными между процессами с помощью JSON:

  1. Создайте пользовательский процесс, в ходе которого соберутся нужные данные для доверенности: её тип, период действия, информация о доверителе, представителе и т. д.

Данные могут поступать через интеграцию с другой системой или заполнение заявки на ранних этапах процесса.

  1. На вкладке Контекст добавьте переменную типа Строка > Текст и заполните её структурой JSON со свойствами, которые соответствуют данным доверенности. Структуры для разных типов доверенностей описаны ниже.
  2. Разместите на схеме блок для запуска подпроцесса Создать МЧД. В нём настроен скрипт для обработки данных, полученных в формате .json, и автоматической генерации доверенности.
  3. Соотнесите переменную Json доверенности дочернего процесса с контекстной переменной родительского процесса.
  4. Добавьте на схему дальнейшие шаги работы с доверенностью или завершите настройку и опубликуйте процесс.  

Таким образом вы создадите доверенность без привлечения сотрудника для этой задачи.

Структуры JSON для создания доверенности

Ниже перечислены структуры для передачи свойств следующих типов доверенностей:

  • B2G501 — доверенность ФНС 5.01;
  • B2G502 — доверенность ФНС 5.02;
  • B2B — доверенность B2B;
  • EMCHD — доверенность EMCHD;
  • SFR — доверенность SFR.

Признак возможности оформления передоверия

Принимает значение:

revocationFlag

Признак утраты полномочий при передоверии

Свойство обязательно при значении свойства poaTransferRestrictions = 1 или 3;

Элемент не заполняется при значении элемента poaTransferRestrictions = 2.

Принимает значение: 

1 — полномочия не утрачиваются в процессе передоверия;

2 — полномочия утрачиваются в процессе передоверия

jointPermissionFlag

Признак совместных полномочий

Принимает значение: 

1 — полномочия индивидуальны;

2 — полномочия совместные

3 — передоверие возможно с последующим передоверием

revocationFlag

Признак утраты полномочий при передоверии

Свойство обязательно при значении свойства poaTransferRestrictions = 1 или 3;

Элемент не заполняется при значении элемента poaTransferRestrictions = 2.

Принимает значение: 

1 — полномочия не утрачиваются в процессе передоверия;

2 — полномочия утрачиваются в процессе передоверия

jointPermissionFlag

Признак совместных полномочий

1 — полномочия индивидуальны;
