Решение «Поиск объектов в документе»

Решение предназначено для поиска объектов на отсканированных документах формата .pdf, .jpg, .png, .tiff и т. д. При помощи искусственного интеллекта в документах распознаются печати, подписи, qr и штрихкоды. Также доступна возможность внедрить распознавание любых пользовательских объектов, используемых в документообороте компании. Зашифрованная информация на найденном объекте будет сохраняться в контекстную переменную. Вы сможете использовать эти данные в ходе бизнес-процессов.  

Кроме того, с помощью решения можно осуществлять потоковое сканирование большого количества документов. В ходе процесса модель ИИ будет разделять документы на отдельные файлы по наличию на страницах распознанных объектов. Затем документы можно скачать на локальный компьютер или сохранить в отдельном приложении для дальнейшей регистрации или архивации.

В состав решения входят:

  • раздел Документы из разделённых pdf с преднастроенным одноимённым приложением. В этом приложении в виде отдельных элементов будут храниться разделённые в ходе бизнес-процесса документы;
  • модуль Поиск объектов в документе. С его помощью вы легко сможете внедрить модель искусственного интеллекта в процессы обработки документов. После подключения модуля в дизайнер бизнес-процессов добавятся графические элементы:
    • Поиск печати, подписи, qr-кода и штрихкода;
    • Разделить документ по штрихкодам;
    • Скачать части разделенного pdf;
    • Сохранить разделенные файлы в приложении.

Установка решения и подключение модуля

Чтобы установить модель искусственного интеллекта в систему, выполните следующие действия:

  1. На главной странице, в левом меню нажмите на значок . В открывшемся окне Настройка Разделов и Решений перейдите на вкладку Решения, нажмите кнопку +Добавить решение и выберите Скачать.
  2. В открывшемся каталоге ELMA365 Store выберите решение Поиск объектов в документе и установите его.

После загрузки решения в систему добавится раздел Документы из разделённых pdf, а также модуль Поиск объектов в документе. Поскольку решение является платным, его необходимо активировать.

  1. Перейдите в раздел Администрирование > Управление лицензиями и напротив названия решения нажмите кнопку Активировать.
  1. Подключите и настройте модуль. Для этого перейдите в раздел Администрирование > Модули > Поиск объектов в документе.

  1. На странице модуля укажите необходимые настройки. При заполнении полей не забудьте сохранять настройки каждый раз, если вы переходите в другие разделы системы:

  • Адрес для обращения — для облачной редакции ELMA365 укажите IP и порт сервера, предоставленные вам при покупке решения Поиск объектов в документе. Для редакции On-Premises указывается адрес сервера, на котором установлена платформа ИИ;
  • Токен — скопируйте и вставьте в это поле сгенерированный ключ токена, созданного в разделе Администрирование > Токены. С его помощью сервису ELMA AI предоставляется доступ к разделу Файлы для сохранения результатов обработки документов;
  • UUID директории — укажите папку в разделе Файлы, куда будут сохраняться документы в ходе работы блока Разделить документы по штрихкодам. Для этого перейдите в нужную папку и в URL-адресе страницы после значения /files/ скопируйте uuid директории. Затем вставьте его в это поле;
  • URL текущей площадки — скопируйте и вставьте в это поле URL-адрес компании, в которую вы установили решение.
  1. Нажмите Сохранить.

Настройки завершены. Теперь в дизайнере бизнес-процессов доступны специальные блоки, которые обрабатывают документы при помощи искусственного интеллекта. Их можно использовать по отдельности или группировать.  

Поиск объектов в документе в ходе процесса

С помощью блока Поиск печати, подписи, qr-кода, штрихкода модель ИИ в ходе процесса может распознавать объекты на загруженном документе. Зашифрованные данные будут сохраняться в контекстную переменную.

Для примера рассмотрим процесс подготовки отчёта по командировке. По окончании рабочей поездки сотруднику назначается задача, в которой он заполняет информацию о командировке. Также он может запросить компенсацию за дополнительные расходы и загрузить чек об оплате с qr-кодом.

Затем с помощью блока Поиск печати, подписи, qr-кода, штрихкода зашифрованные данные о совершенной покупке считываются и сохраняются в контекстную переменную. На основании полученных данных формируется отчёт о командировке. Бухгалтеру назначается задача ознакомиться с отчётом, проверить данные по чеку и согласовать выплату денег сотруднику.

Разделение файла по штрихкодам и сохранение его частей в ходе процесса

В ходе бизнес-процесса можно разделить пакет отсканированных документов на несколько файлов по условию наличия штрихкода на страницах. Затем части разделённого файла можно загрузить на локальный компьютер или сохранить в виде отдельных элементов приложения.

Например, в компании применяется потоковое сканирование договоров, и на первой странице каждого из них присутствует штрихкод. Отсканированный пакет документов необходимо разделить на отдельные файлы.

После запуска экземпляра процесса пользователь получает задачу, на форму которой загружает исходный файл формата .pdf. Затем с помощью блока Разделить документ по штрихкодам модель ИИ распознаёт, на каких страницах добавленного файла присутствует штрихкод и разделяет его на несколько документов. Части разделённого файла сохраняются в файлах компании, в папке, указанной в настройках модуля Поиск объектов в документе.

Далее процесс доходит до блока Скачать части разделенного pdf, и исполнителю назначается задача ознакомления с результатами разделения файла. На её форме он увидит: файл формата .pdf со страницами договора с штрихкодом, поля с зашифрованной в штрихкоде информацией, а также файл с остальными страницами договора. Файлы можно открыть в режиме просмотра или скачать на локальный компьютер.

Когда процесс дойдет до графического элемента Сохранить разделённые файлы в приложении, исполнителю назначаются задачи для каждого файла, полученного в результате разделения. Сотрудник может ознакомиться с файлом и зашифрованной информацией на найденном штрихкоде. Затем он может сохранить документ в виде элемента приложения Документы из разделённых pdf, которое добавляется в систему при установке решения Поиск объектов в документе. В этом случае пользователи смогут работать с разделёнными файлами как с элементами приложений типа Документ, например отправлять их на ознакомление или согласование.

Схема процесса будет выглядеть следующим образом:
