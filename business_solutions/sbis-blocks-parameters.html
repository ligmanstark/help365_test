Входные и выходные атрибуты блоков решения «СБИС»

Перед началом настройки блока из решения СБИС добавьте в контекст процесса переменные, соответствующие входным и выходным атрибутам блока. В настройках каждого блока:

  • входные атрибуты — сопоставьте с переменными, которые содержат данные из ELMA365 для их передачи в СБИС;
  • выходные атрибуты — укажите переменные, в которые будет записан результат, полученный из СБИС.

Чтобы упростить настройку, рекомендуем для переменных использовать названия, совпадающие с названиями атрибутов.

Название входного атрибута

Тип сопоставляемой переменной. Комментарии

Юр.  лицо

Приложение (один). Обязательный параметр.

Переменная ссылается на системный справочник Мои юридические лица. Записанное в неё юрлицо должно быть подключено к сервису СБИС в настройках модуля. От этого юрлица будут выполняться действия в СБИС.

Элемент приложения

Приложение (один). Обязательный параметр.

Переменная ссылается на приложение типа Документ, для которого настроено соответствие документов в настройках модуля.

ИНН контрагента

Строка. Обязательный параметр.

Содержит ИНН контрагента, которому будет направлен документ.

КПП контрагента

Строка. Обязательный параметр.

Содержит КПП контрагента, которому будет направлен документ.

Тип документа

Строка (текст). Обязателен для заполнения в блоке СБИС записать документ. Содержит тип документа в СБИС. Подробнее о типах смотрите в официальной документации сервиса.

Номер документа

Строка. Если переменная не указана, будет передано значение согласно настройкам сопоставления атрибутов в модуле либо пустое значение.

Дата документа

Строка в формате ДД.ММ.ГГГГ.

Если переменная не указана, будет передано значение согласно настройкам сопоставления атрибутов в модуле либо пустое значение.

Подпись

Файлы. Обязательный параметр

ID вложения

Строка. Содержит идентификатор вложения СБИС.

Необязательный атрибут. ID можно сгенерировать вручную в виде uid-номера и передать в СБИС. Если ID не указан, он сгенерируется автоматически при сохранении вложения в СБИС.

Вложение

Файлы. Обязательный параметр

Идентификатор доверенности

Строка. Обязателен для заполнения в блоке СБИС импорт доверенности.

Содержит идентификатор МЧД, который должен быть предварительно импортирован в личный кабинет СБИС и иметь статус Действует

ИНН доверителя

Строка. Обязательный параметр

Отпечаток сертификата

Строка. Содержит отпечаток добавленного в СБИС сертификата для подписания документа.

Условно-обязательный параметр: требует заполнения, если не сопоставлены ФИО, ИНН и Должность.

ФИО

Строка. Условно-обязательный параметр: требует заполнения, если не сопоставлен Отпечаток сертификата.

Содержит данные Ф. И. О. подписанта.

ИНН

Строка. Условно-обязательный параметр: требует заполнения, если не сопоставлен отпечаток сертификата.

Содержит ИНН подписанта

Должность

Строка. Условно-обязательный параметр: требует заполнения, если не сопоставлен отпечаток сертификата.

Содержит должность подписанта.

Этап (строка)

Строка. Необязательный параметр.

Позволяет настроить повторное выполнение блока СБИС записать и отправить документ с определённого этапа, на котором произошла ошибка и значение которого будет передано в блок. Если переменная не сопоставлена, выполнятся все три этапа: записать документ, записать вложение, отправить документ.

Комментарий

Строка (текст). Обязателен для заполнения в блоках:

  • СБИС подготовить аннулирование;
  • СБИС выполнить аннулирование;
  • СБИС подготовить отказ от аннулирования;
  • СБИС выполнить отказ от аннулирования;
  • СБИС выполнить подтверждение аннулирования.
