Аудит изменений платформы

Бизнес-решение позволяет пользователям отслеживать изменения данных в любом приложении ELMA365.

События создания, изменения, удаления и восстановления элемента приложения сохраняются в его ленту. Эти данные можно отобразить также в карточке элемента на отдельной вкладке. Каждая запись содержит информацию об авторе и дате события. Если изменения вносятся в поле элемента, можно просмотреть предыдущее и новое значения этого поля.

Отчёт обо всех событиях, связанных с элементом, можно сохранить в файл универсального для табличных данных формата .csv и проанализировать работу с элементом.

начало внимание

Решение Аудит изменений платформы лицензируется. Вы можете ознакомиться с ним, установив на триальный период. Также для корректной работы установите бизнес-решение Аудит изменений CRM.

конец внимание

Решение Аудит изменений платформы:

  • содержит модуль Аудит изменений платформы. В его настройках администратор системы выбирает приложения, изменения которых будут записываться, устанавливает срок хранения данных и набор отслеживаемых событий;
  • использует следующие компоненты решения Аудит изменений CRM:
      • раздел Аудит изменений — содержит приложение Изменение элемента, доступное администратору системы. В нём в виде отдельных записей сохраняются все события, связанные с элементами приложений;  
      • виджет История изменений — добавляется на формы приложений и позволяет пользователям просматривать события в карточках элементов этих приложений.

Первичная настройка бизнес-решения «Аудит изменений платформы»

  1. Из каталога ELMA365 Store импортируйте:
    • решение Аудит изменений CRM;
    • платное решение Аудит изменений платформы на триальный период.

начало внимание

По окончании триального периода платное решение Аудит изменений платформы нужно активировать.

конец внимание

  1. В разделе Администрирование > Модули:
    • выключите модуль Аудит изменений CRM. Это нужно для корректной работы решения Аудит изменений платформы;
    • включите модуль Аудит изменений платформы.

  1. Выполните настройки на странице модуля Аудит изменений платформы:

    • Время хранения изменений — установите, сколько дней записи об изменениях будут храниться в разделе Аудит изменений и в карточках элементов. Выберите 7 или 14 дней. Для редакции SaaS Enterprise и поставки On-Premises можно также выбрать срок 30, 90, 180 дней или вариант Бессрочно. По истечении этого времени записи автоматически удалятся;
    • Логируемые приложения — выберите приложения, изменения в которых нужно хранить. Доступны как системные, так и пользовательские приложения. Если вы уже использовали решение Аудит изменений CRM, заново выберите приложения CRM для записи изменений и добавьте любые другие приложения. Обратите внимание, для всех данных будет применяться срок хранения, который указан в модуле Аудит изменений платформы;
    • Логируемые события — укажите события, которые нужно фиксировать для элементов выбранных приложений;
    • Уведомлять о событиях — обозначьте события, о которых вы хотите получать оповещения в ленту элемента.
  1. Сохраните настройки.
  2. Перезагрузите страницу. Это нужно для корректной работы опции Посмотреть подробнее в карточке элемента.
  3. На формы выбранных приложений добавьте виджет История изменений.
  4. Настройте группы пользователей и назначьте им права доступа к данным раздела Аудит изменений. Это выполняется так же, как при настройке решения Аудит изменений CRM.

Настройки бизнес-решения Аудит изменений платформы завершены.

Теперь пользователи могут просматривать историю событий в карточке элемента так же, как при работе с решением Аудит изменений CRM. Подробнее об этом читайте в статье «Работа с аудитом изменений CRM».
