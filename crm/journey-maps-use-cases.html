Примеры сценариев для карты коммуникаций

Кейс 1. Приглашение на вебинар

Настроим отправку приглашения на вебинар подписчикам выбранного сегмента, которые указаны в лидах. Письмо отправляется клиенту после того, как статус лида изменится на Обработка. Также настроим отправку напоминания о вебинаре, если клиент открыл первое письмо.

Шаги сценария:

  1. Выберите сегмент подписчиков.
  2. В триггере Ожидание изменения статуса задайте изменение статуса лида на Обработка.
  3. В блоке Ожидание события укажите дату отправки приглашения на вебинар.
  4. В блоке E-mail настройте отправку письма с приглашением на вебинар.
  5. В шлюзе Вовлечённость укажите блок E-mail Приглашение на вебинар и включите опцию Открытие письма. Также задайте время ожидания пять дней и переход по умолчанию к конечному событию. Если в течение пяти дней клиент не откроет письмо, сценарий завершится.
  6. В блоке Ожидание события укажите дату отправки напоминания.
  7. Настройте отправку письма с напоминанием о вебинаре. Его получат только те клиенты, которые открыли первое письмо.

Кейс 2. Отзывы после покупки

Настроим отправку письма с просьбой оставить отзыв после успешного завершения сделки.

Шаги сценария:

  1. Выберите сегмент подписчиков.
  2. В триггере Ожидание изменения статуса задайте изменение статуса сделки на значение Закрыто успешно.
  3. В блоке Таймер укажите период ожидания 14 дней.
  4. В блоке E-mail настройте отправку письма с запросом оставить отзыв.

Кейс 3. Акция Золотые клиенты

Настроим отправку письма клиентам в случае, когда тип клиента изменяется на значение Золото. В этом кейсе предположим, что в карточку приложения Компании добавлено свойство Тип клиента со значениями Бронза, Серебро, Золото. Письмо будет отправлено подписчику, указанному в карточке компании.

Шаги сценария:

  1. Выберите сегмент подписчиков.
  2. В триггере Ожидание значения поля задайте изменение свойства Тип клиента для элемента приложения Компании на значение Золото.
  3. В блоке Таймер укажите период ожидания 1 день.
  4. В блоке E-mail настройте отправку письма с акцией для золотых клиентов.

Кейс 4. Рассылка по продуктам

Настроим отправку писем новым лидам, заинтересованным в определённом продукте компании.

Шаги сценария:

  1. Выберите сегмент подписчиков.
  2. В триггере Создание элемента приложения задайте добавление нового элемента приложения Лиды с подписчиком сегмента.
  3. В блоке Таймер укажите период ожидания 1 день.
  4. В шлюзе ИЛИ настройте условия для каждого перехода:
    • если в карточке лида указан продукт А, сценарий перейдёт к блоку E-mail по продукту А;
    • если в карточке лида указан продукт В, сценарий перейдёт к блоку E-mail по продукту В;
    • в любом другом случае выполнится переход, заданный по умолчанию, и рассылка не выполнится.
  1. В блоках E-mail настройте отправку писем по соответствующим продуктам компании.

Кейс 5. АВ-тест для email-рассылки

Для оценки двух вариантов маркетингового письма можно настроить email- рассылку с новой акцией. Выберите подписчиков, по которым не совершались активные действия за последнюю неделю. С помощью шлюза АВ-тест разделите подписчиков на две равные части, каждая из которых получит свой вариант письма. После выполнения сценария можно просмотреть статистику по рассылкам в отчёте Вовлечённость.

Шаги сценария:

  1. Выберите сегмент подписчиков.
  2. В триггере Отсутствие изменения статуса выберите приложение Лиды и статус Обработка. Также задайте срок отсутствия изменения статуса 7 дней.
  3. В шлюзе АВ-тест установите значение 50% для каждой части потока подписчиков.
  4. В блоках E-mail настройте отправку разных вариантов писем. Одна половина подписчиков получит письмо А, а другая – письмо В.
