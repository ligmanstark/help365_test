Маркетинг

Раздел Маркетинг является частью системного решения ELMA365 CRM и используется для отправки рассылок на электронную почту и управления клиентской базой из интерфейса ELMA365.

С помощью раздела Маркетинг вы можете:

  • осуществлять массовые и триггерные рассылки писем отдельным сегментам клиентов, например, оповещать об изменении цен на продукты или услуги, акциях, вебинарах и т. д.;
  • реализовать долгосрочную поддержку клиентов: отправлять новости, сообщать об обновлениях и т. д.;
  • формировать сегменты клиентов на основе данных о контактах из любых приложений системы, а также из внешних систем с помощью методов API;
  • создавать полноценные шаблоны писем, используя встроенный email-редактор;
  • определять активность клиентов и их заинтересованность в продукте или услуге.

Для отправки писем используется сторонний сервис рассылки. Вы можете подключить готовый или настроить пользовательский модуль интеграции. Через него данные поступят в сервис рассылки, который отправит письма на электронную почту клиентов.

Начало внимание

Раздел доступен при активации одного из платных решений CRM, в которые включены инструменты для ведения маркетинговых кампаний. Работать в нём смогут только пользователи с лицензией такого решения.

Конец внимание 

Как организован раздел

Раздел включает в себя:

  • приложение Категории предпочтений — хранит список тематических категорий для рассылок клиентам;
  • страницу Подписчики — здесь отображаются все клиенты, давшие согласие на получение рассылок;
  • приложение Сегменты подписчиков — позволяет сформировать сегменты клиентов, отфильтрованные по каким-либо критериям;
  • страницу Карты коммуникаций — здесь можно создавать триггерные рассылки, которые выполняются по заданному сценарию и запускаются по настроенному триггеру. Обратите внимание, страница доступна только в поставке On-Premises;
  • приложение Массовые рассылки — позволяет создать рассылки для отправки писем в запланированное время всем подписчикам выбранного сегмента;
  • приложение Шаблоны писем — позволяет создавать шаблоны писем, используемые в рассылках, а также импортировать готовые шаблоны из подключённого сервиса рассылки;
  • отчёт Вовлечённость — здесь можно посмотреть статистику по каждой отправленной массовой или триггерной рассылке.

Первичная настройка раздела

Прежде чем приступать к работе с разделом, выполните следующие настройки:

  1. Подключите модуль интеграции с сервисом рассылки. Вы можете:
  • подключить готовый модуль интеграции с одним из следующих сервисов:
    • Unisender — для выполнения массовых рассылок клиентам. Обратите внимание, этот модуль не применяется для отправки триггерных рассылок;
    • Unisender Go или Sendsay — для реализации как массовых, так и триггерных рассылок. Обратите внимание, эти модули не совместимы с редакцией Saas Standard;
  • или настроить пользовательский модуль интеграции с другим сервисом рассылки, используя методы API. Такой модуль позволит выполнять как массовые, так и триггерные рассылки, доступные в поставке On-Premises. Подробнее о настройке интеграции читайте в справке ELMA365 TS SDK.
  1. Выберите приложения из раздела CRM, из которых будет осуществляться выборка подписчиков.
  2. Задайте правило, по которому будет определяться, как клиенты с совпадающими email будут становиться подписчиками.
  3. Создайте шаблоны писем с помощью встроенного email-редактора или импортируйте их с подключённого сервиса рассылок.
  4. Создайте категории предпочтений — список тематик для рассылок, на получение которых будут давать согласие пользователи.
  5. Создайте базу подписчиков, по которым будет осуществляться рассылка.

Работа с разделом

Ответственный сотрудник выполняет следующий порядок действий в настроенном разделе:

  1. Сформируйте сегменты подписчиков из готовой базы клиентов. Подписчики, входящие в сегмент, будут получать рассылку писем на электронную почту. Сегменты формируются по каким-либо критериям. Например, можно создать сегмент новых клиентов, чтобы разослать им приглашение на обучающий вебинар.
  2. Создайте в email-редакторе или импортируйте шаблоны писем, которые будете отправлять подписчикам.
  3. Затем создайте рассылки следующих типов:
    • массовая — отправляет письма в запланированное время всем подписчикам выбранного сегмента;
    • триггерная — этот тип рассылки можно использовать в поставке On-Premises. Такая рассылка выполняется по сценарию, заданному в карте коммуникаций.

Начало внимание

Работа с картами коммуникаций доступна только в поставке On-Premises.

Конец внимание 

  1. Статистика по каждой массовой или триггерной рассылке доступна в отчёте Вовлечённость. Здесь вы увидете данные по отправленным письмам, а также количество переходов по ссылкам из писем.
  2. Определить активность определённого контакта можно в его карточке с помощью виджета Рассылки, в котором также отображается статистика рассылок.

Таким образом вы сможете создавать сегментированные группы подписчиков из базы контактов и осуществлять быструю рассылку писем из интерфейса ELMA365.
