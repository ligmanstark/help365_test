Присвоение клиенту стадии прогрева

В ELMA365 сотрудники отдела продаж совершают различные действия для коммуникации с клиентом. При планировании этих действий пользователи указывают определённый контакт, с которым необходимо связаться. Все активности, совершённые с контактом, фиксируются в системе.

По умолчанию отслеживаются следующие действия:

  • Входящие — действия, которые совершил контакт: открыл письмо‑рассылку, перешёл по вложенной ссылке в письме. Также к входящим относятся действия пользователей в ELMA365: создание лида, сделки, выполнение задачи CRM;
  • Исходящие — действия, совершённые пользователем в ELMA365: создание задачи CRM, отправка письма из встроенного почтовика.

Администратор системы может создать пользовательские действия, например, чтобы отслеживать дополнительные способы связи с клиентом или его действия на сайте компании. Записи о пользовательских активностях фиксируются в системе в ходе бизнес‑процесса. Подробнее читайте в статье «Настройки истории активностей».

Записи об активностях отображаются в карточках контактов в виджете История активностей. Это позволяет менеджеру по продажам легко отслеживать действия с контактом. Также данные по взаимодействиям с клиентом используются для определения его готовности к покупке, то есть стадии прогрева.

начало внимание

Присвоение стадий прогрева доступно при активации одного из платных решений CRM, в которые включена фиксация истории активностей. Работать со стадиями могут только пользователи с лицензией такого решения.

конец внимание

По умолчанию в разделе преднастроены следующие стадии:

  • Новый — контакт без активностей, с которым ещё не связывались менеджеры;
  • Холодный — с контактом совершенно до двух активностей, например, создана сделка и запланирован звонок;
  • Теплый — по контакту произведено от трёх до шести действий; 
  • Горячий — проработанный контакт с семью и более совершёнными активностями.

Администратор системы может отредактировать названия стадий и условия их присвоения.

Для просмотра текущего прогресса используется виджет Стадия прогрева. Он отображается в карточке контакта. Администратор может добавить виджет на формы других приложений, например, Сделки или Компании. 

Если в элементе приложения указано несколько контактов, в карточке будет отображаться стадия прогрева для каждого из них.  

Изменить стадии прогрева

Вы можете отредактировать названия заданных по умолчанию стадий, добавить новые этапы, выбрать активности, по которым производится расчёт и т. д.

начало внимание

Задать настройки стадий прогрева могут только пользователи, входящие в группу Администраторы.

конец внимание

  1. Перейдите к настройкам стадий прогрева. Это можно сделать двумя способами:
    • откройте раздел Администрирование > Стадии прогрева;
    • откройте раздел CRM. Рядом с названием раздела нажмите значок шестерёнки и выберите пункт Настройка стадий прогрева.
  2. На открывшейся странице настройки выполняются на двух вкладках: Основное, Пересчет.

  1. В поле Активности раскройте выпадающий список или нажмите Показать, чтобы посмотреть, по каким действиям рассчитываются стадии прогрева на текущий момент. 

Выберите действия, которые нужно учитывать. Например, если вы не хотите, чтобы стадия прогрева менялась при создании задач CRM, отключите исходящие активности Задача «Письмо» создана, Задача «Звонок» создана и т. д.

Обратите внимание, эти настройки не влияют на отображение записей в виджете История активностей. Скрыть активность из виджета можно в настройках истории активностей.

  1. Сохраните список, нажав Применить.

  1. В разделе CRM > Настройка стадий прогрева задайте степени заинтересованности клиента. Вы можете изменить названия существующих стадий прогрева или создать новые. Для этого нажмите + Добавить стадию и укажите её название в поле. Запись добавится перед финальным этапом. 
  2. Для удаления стадии напротив её названия нажмите значок корзины. Обратите внимание, первый и последний этап удалить нельзя.  

  1. В столбце Сумма активностей укажите, сколько действий необходимо совершить пользователю для присвоения каждой стадии прогрева. При выборе нового значения показатель следующей стадии автоматически пересчитывается. 

  1. Сохраните настройки на странице. 
  2. При добавлении новых стадий прогрева или изменении суммы активностей необходимо применить условия присвоения этапов. Для этого перейдите на вкладку Пересчет. Напротив каждой стадии указывается её текущий статус. Если для стадии не изменены условия присвоения, например, отредактировано только её название, вы увидите статус Пересчитана.
  3. Внизу страницы нажмите кнопку Пересчитать измененные и дождитесь окончания процесса. 

Настройки завершены. Теперь стадии будут присваиваться по новым условиям. 

Изменённые настройки стадий прогрева можно загрузить в другую компанию. Для этого экспортируйте раздел CRM и обновите его в другой версии ELMA365.  

  1. Чтобы пользователи могли просматривать текущий этап, добавьте на формы приложений CRM виджет Стадия прогрева.
